Everything you need to stay secure

Built for operators who take security seriously.
AES-256 Encryption
Military-grade symmetric encryption locks every file with a unique 256-bit master key. Impossible to brute-force.
LSB Steganography
Hide encrypted keys inside ordinary images using Least Significant Bit encoding. The key is invisible to the naked eye.
Live Permission Control
Approve or deny decryption requests in real-time from anywhere using Firebase-powered cloud permissions.
Multi-Operator Support
Register multiple operators with individual accounts. Data isolation ensures each operator only sees their own packages.
Burn After Read
Automatically destroy permission records after a message is extracted. Zero evidence left behind.
Offline-First Architecture
SQLite-powered local authentication means the app works fully offline. No cloud dependency for core operations.

How Sentinel Vault Works

1
Encrypt
Select your target folder. Sentinel generates a master key and locks every file with AES-256 encryption.
2
Hide the Key
The master key is invisibly embedded into a cover image using LSB steganography. The image looks unchanged.
3
Share Securely
Send the stego-image and a Share ID to your recipient via any channel. Neither alone can unlock the files.
4
Approve & Decrypt
The recipient requests access. You approve in real-time. Their Sentinel client downloads and decrypts everything automatically.
ZERO-KNOWLEDGE ARCHITECTURE

No central server ever sees your data.

Sentinel Vault was architected around a fundamental principle: if there's nothing to steal, there's nothing to hack. Your files never leave your machine. Encryption happens locally. The cloud only holds permission IDs — never the keys, never the data.

Files are encrypted locally before any network activity occurs
Master keys are hidden in images — never stored in any database
Firebase only stores permission status, not encrypted content
